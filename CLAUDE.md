# Agent Guide

This file provides guidance to AI coding agents when working with code in this repository.

## What this repository is

This is **telepresence.io** — the Docusaurus 3 static documentation/marketing site served at https://telepresence.io. It is *not* the Telepresence application source. The application code lives in a separate repo: `telepresenceio/telepresence`. The actual product docs are authored there (under its `docs/` directory) and pulled into this repo as versioned snapshots.

## Package manager: yarn, not npm

Always use `yarn`. `.npmrc` + the `engines` field in `package.json` are configured to make `npm install` fail on purpose. Running npm risks a broken build or accidentally committing `package-lock.json`. Node version is pinned in `.node-version` (v20.12.2).

## Commands

```bash
yarn                 # install dependencies
yarn start           # hot-reloading dev server (docusaurus start)
yarn build           # production build into ./build
yarn serve           # serve the production build
yarn typecheck       # tsc (no emit)
yarn run eslint .    # lint
yarn clear           # clear docusaurus cache (use when builds behave oddly)
```

There is no test suite. CI quality gates (`.github/workflows/lint.yml`) are: `yarn build`, a **dirty check** (the build must not modify any tracked files — `git status --porcelain` must be empty after build), and `eslint`. The build fails on broken internal links (`onBrokenLinks: 'throw'`). A separate workflow (`blc.yaml`) runs an external broken-link checker. Deployment is handled by Netlify from `master` (Netlify also creates PR preview deploys); the GitHub workflows do not deploy.

## Pulling docs from the telepresence repo

Versioned docs are generated from the `telepresenceio/telepresence` git repo via the `Makefile`, which adds that repo as a `telepresence` git remote.

```bash
# Regenerate a minor version's docs from a branch of the telepresence repo.
# read-branch checks out docs/ from telepresence/<DOCS_BRANCH> into ./docs,
# then docusaurus snapshots it into versioned_docs/version-<DOCS_VERSION>.
DOCS_BRANCH=release/v2.29 DOCS_VERSION=2.29 make generate-version

make drop-version DOCS_VERSION=2.21   # remove a version entirely
make pull-docs                        # snapshot all release tags that lack a versioned_docs dir
```

`./docs/` is a transient working directory used only during generation — `generate-version`/`pull-docs` restore it from HEAD when done. Do not author docs there. The committed source of truth for each published version is `versioned_docs/version-<X.Y>/`, listed in `versions.json`, with sidebars in `versioned_sidebars/`.

## How the docs system works (the non-obvious parts)

- **Sidebar** is *not* generated from the file tree. A custom `sidebarItemsGenerator` in `docusaurus.config.ts` reads each version's `doc-links.yml` to build the sidebar. A doc file with no entry in `doc-links.yml` produces a build warning (except `common/*`). When adding/removing a doc page, update that version's `doc-links.yml`.
- **Variable substitution**: a markdown `preprocessor` in `docusaurus.config.ts` replaces `$var$` tokens using the `variables.yml` in the same version directory (e.g. `$version$` → `2.29.1`). This only applies inside `versioned_docs/version-*/`.
- **Release notes** (`versioned_docs/version-*/release-notes.mdx`) are **generated by `relnotesgen` — DO NOT EDIT**. They use the `<Note>`/`<Title>`/`<Body>` React components from `src/components/ReleaseNotes`. `<Title type=...>` accepts `bugfix`, `change`, `feature`, or `security` (each maps to an icon).
- **GitHub-style admonitions** (`> [!NOTE]` etc.) are converted via `remark-github-admonitions-to-directives` for both docs and blog.
- `editUrl` for docs points back to `telepresenceio/telepresence` (`release/v2/docs/...`), because that is where docs are actually edited.

## Site (non-docs) structure

- `src/pages/**` — standalone pages (home `index.tsx`, `about`, `community`, `case-studies/*`, `404.tsx`).
- `src/components/**` — shared React components (e.g. `ReleaseNotes`, `Platform`, `InterceptAnimation`).
- `src/theme/**` — Docusaurus swizzled theme overrides (navbar, sidebar, search, MDX components, color mode).
- `src/plugins/configure-svgo.ts` — custom plugin; `*.inline.svg` assets are imported as React components.
- `src/css/custom.css` + `src/css/telepresence.scss` — global styles (Sass via `docusaurus-plugin-sass`).
- `blog/` — release announcement posts; authors in `blog/authors.yml`.
- `static/` — served as-is, including Netlify `_redirects` and `robots.txt`.

## Conventions

- `.editorconfig` governs indentation/whitespace; ensure your editor honors it.
- ESLint config is `.eslintrc.js`; TS config is `tsconfig.json`.
